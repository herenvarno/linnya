Chapter 1. 命名规则

MusicMe的命名原则是“说明功能、体现结构、保持简单”

在函数或变量的命名中，必须体现出该函数或变量的功能作用、生命周期、使用权限、定义位置、所属模块等信息。并且名字必须保持简单明了，要善于使用约定俗成的简写以提高编码效率。另外，为了避免出现记忆错误，函数和变量的名字全部使用小写，并以下划线分割单词，但是宏名应全部为大写，以示区别。

基于以上的规则，我们制定出以下的命名方案：

(前缀)_(类型)_(模块)_(功能)

前缀

在MusicMe的核心源代码中，每一个函数或变量都应拥有一个相同的前缀——mm，表示MusicMe。

类型

类型标示了名称所代表的东西的类型，分为变量、函数和自定义类型。

类型的定义规则：

  1. var - 代表变量

  2. func - 代表函数

  3. type - 代表自定义类型

  4. MACRO - 代表宏

模块

模块表示了一个名字的定义位置和所属模块，一般使用模块的缩略形式。

各个内核模块的简写形式：

  1. 消息传递模块 - msg

  2. 配置寄存模块 - conf

  3. 数据库模块 - db

  4. 音频控制模块 - audio

  5. 键盘绑定模块 - key

功能

功能部分是用来描述此变量或函数的具体作用的英文单词或词组，也是各个变量的相互区别的字符串。它没有统一的定义规则，只要是能表达清除变量/函数的功能意义即可。

示例：

  1. MM_MACRO_CONF_PATH 在conf模块中定义的表示路径的宏

  2. mm_var_conf_table 在conf模块中定义的指向配置哈希表的指针变量

  3. mm_func_conf_put 在conf模块中定义的写入一个配置的函数

  4. mm_type_conf_element 在conf模块中定义的配置元素结构类型
