MusicMe的命名原则是“说明功能、体现结构、保持简单”

在函数或变量的命名中，必须体现出该函数或变量的功能作用、生命周期、使用权限、定义位置、所属模块等信息。并且名字必须保持简单明了，要善于使用约定俗成的简写以提高编码效率。

基于以上的规则，我们制定出以下的命名方案：

宏的命名：

在MusicMe的核心源代码中，宏全部使用大写字母表示，其结构组成为：MM_(模块)_(功能)

自定义结构体类型的命名：

自定义结构体是一类封装了的变量类型，他的名字应该和默认的类型命名相类似，结构为：mm(模块)(功能)。需要注意的是，模块和功能的单词首字母要大写，以区分不同的单词。

变量的命名：

变量的命名结构为：mm_(模块)_(功能（名词）)。请注意，变量的功能描述部分一般使用名词。

函数的命名：

函数的命名方式和变量一样，只是最后的功能描述部分一般使用动词或动词短语。

示例：

  1. MM_CONF_PATH 在conf模块中定义的表示路径的宏

  2. mm_conf_table 在conf模块中定义的指向配置哈希表的指针变量

  3. mm_conf_put 在conf模块中定义的写入一个配置的函数

  4. mmConfElement 在conf模块中定义的配置元素结构类型
